> 记录在WSL Ubuntu中安装CUDA时，因`apt-key`命令被弃用而报错，并通过NVIDIA官方keyring包顺利解决的经历。

## 问题现象

我在WSL（Windows Subsystem for Linux）中安装CUDA Toolkit，按照NVIDIA官方文档的指引，添加了CUDA本地仓库并尝试导入GPG密钥：

```bash
sudo apt-key add /var/cuda-repo-wsl-ubuntu-11-6-local/7fa2af80.pub
```
然而终端却返回了错误信息：
```bash
sudo: apt-key: command not found
```
apt无法验证仓库签名，后续的sudo apt-get update也因此失败，CUDA安装卡在了第一步。

## 原因分析
apt-key命令在较新版本的Ubuntu（22.04及之后）中已被弃用并移除。我使用的WSL发行版是Ubuntu 22.04，自然没有这个命令。

NVIDIA官方曾经推荐使用apt-key来添加公钥，但随着Debian/Ubuntu新政策的推行，这种做法已经过时。新的标准做法是将GPG密钥以.gpg文件的形式存放在/usr/share/keyrings/目录，并在源列表（.list）中通过signed-by选项显式指定密钥文件，从而避免使用已废弃的全局apt-key。

## 解决方法
NVIDIA官方提供了自动化keyring包（cuda-keyring），可以一键完成密钥配置，这正是我采用的方案。

### 操作步骤
1.下载keyring包（针对WSL版本）：
```bash
wget https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/cuda-keyring_1.0-1_all.deb
```
该包会自动处理仓库签名密钥的安装。

2.安装keyring包：
```bash
sudo dpkg -i cuda-keyring_1.0-1_all.deb
```
安装过程会将NVIDIA的公钥添加到/usr/share/keyrings/，并更新APT源配置。

3.更新软件包列表：
```bash
sudo apt-get update
```
此时不再报错，APT能正常验证仓库签名。

4.继续安装CUDA Toolkit：
```bash
sudo apt-get -y install cuda
```
安装顺利完成。

整个过程非常顺畅，无需手动处理密钥文件或修改源列表。

### 其他备选方案
如果keyring包无法下载或安装失败，也可以手动导入公钥（使用gpg命令）并编辑源文件，但操作稍显繁琐。对于绝大多数用户，方案一最为简单可靠。

## 关键要点
- 不要在WSL中安装NVIDIA显卡驱动，只需安装CUDA Toolkit，WSL会利用Windows宿主机上的驱动。
- 如果你正在使用较旧的CUDA版本（如11.x），但系统是Ubuntu 22.04+，强烈建议优先使用cuda-keyring包，而非依赖已过时的apt-key。
- 若遇到类似“command not found”错误，应检查操作是否匹配当前系统的工具链，及时采用官方新方案。

## 总结
- 旧版安装教程中的apt-key命令在新版Ubuntu中已不可用，导致CUDA安装报错。
- 通过下载并安装NVIDIA官方的cuda-keyring包，可以自动配置好GPG密钥，完美解决该问题。
- 只需三步（下载、安装、更新），即可绕过apt-key缺失的障碍，继续完成CUDA安装。

小小的工具链变更，却可能让经验丰富的开发者也会卡壳。记下这次排查过程，希望对在WSL中搭建CUDA环境的你有所帮助。
