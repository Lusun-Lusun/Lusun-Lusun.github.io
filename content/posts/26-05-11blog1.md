## 📌 背景与动机
在申请某个证券管培生的过程中，我意识到券商正处于数字化转型的关键期。投研部门每天面临海量的研报与公告，信息过载是行业痛点。为此，我决定开发一个 **AI Agent (智能助理)**，利用大语言模型（LLM）实现对 PDF 研报的深度解析与精准问答。

---

## 🛠️ 第一阶段：架构重构——从“黑盒”走向“透明”

在起步阶段，我遭遇了最大的技术挑战：由于 **LangChain 框架在 2026 年已全面演进至 0.3+ 版本**，传统的 `RetrievalQA` 链式调用频繁报出 `ModuleNotFoundError`。

### 技术洞察
*   **问题核心**：旧版 API 已被废弃，底层架构进行了深度模块化拆分。
*   **解决方案**：放弃“全家桶”式的导入，改用更现代、更稳健的 **LCEL (LangChain Expression Language)**。
*   **核心逻辑**：
    不再依赖预置的黑盒函数，而是通过“管道符（`|`）”手动搭建数据流：
    ```python
    # 2026年主流的 LCEL RAG 架构
    rag_chain = (
        {"context": retriever | format_docs, "question": RunnablePassthrough()}
        | prompt
        | llm
        | StrOutputParser()
    )

---

## 🐛 第二阶段：环境攻坚——解决“消失的路径”
开发过程中，我遇到了严重的“环境污染”问题：即便安装了包，系统依然无法识别。这让我深刻理解了工程标准化的重要性。

### 解决路径
*   **风险隔离**：通过 python -m venv venv 建立独立的虚拟环境，确保项目依赖的纯净性。
*   **路径溯源**：查阅 LangChain 官方迁移指南，意识到核心逻辑已迁移至 langchain-core。
*   **标准化部署**：配置 requirements.txt 和 .gitignore，实现了从开发环境到云端部署的无缝对接。

---

## 🏗️ 第三阶段：功能实现与金融逻辑对齐
作为一个面向金融业务的项目，我特意在技术实现中植入了合规与严谨性：
*   **强制溯源**：在 Prompt 中要求 Agent 必须标注信息来源页码，确保结论可追溯。
*   **语义检索 (RAG)**：利用 ChromaDB 向量数据库，将研报切片并索引，有效解决了 LLM 的“幻觉”问题。
*   **交互界面**：基于 Streamlit 构建了简洁的 Web 页面，实现了“上传-索引-问答”的闭环体验。
