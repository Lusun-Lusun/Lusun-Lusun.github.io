> 记录我在Windows上通过WSL安装Ubuntu并配置CUDA开发环境的完整过程，包括遇到的几个典型问题及解决方法。

## 背景

我的电脑是Windows系统，因为想学习CUDA编程，所以通过WSL（Windows Subsystem for Linux）安装了Ubuntu 26.04 LTS。WSL的好处是可以直接在Windows上运行Linux环境，而且GPU可以透传，方便进行CUDA开发。

按照计划，我需要在Ubuntu中安装CUDA Toolkit，这样才能用`nvcc`编译CUDA程序。下面就是我一步步配置的过程，中间遇到了一些小波折，最终顺利搞定。

## 第一步：已有的WSL环境

其实我已经装好了WSL，直接在PowerShell里用`wsl --install`会提示已经存在，所以我直接用`wsl -d Ubuntu`进入了系统。

进入后，系统显示是Ubuntu 26.04 LTS，内核是WSL2的标准版本。

## 第二步：尝试安装旧版CUDA（失败）

一开始，我手头有一个旧的CUDA 11.6的本地deb包（`cuda-repo-wsl-ubuntu-11-6-local_11.6.0-1_amd64.deb`）。我直接用`sudo dpkg -i`安装，包本身装上了，但最后提示：
```bash
The public CUDA GPG key does not appear to be installed.
To install the key, run this command:
sudo apt-key add /var/cuda-repo-wsl-ubuntu-11-6-local/7fa2af80.pub
```

也就是说，需要添加GPG公钥来验证这个仓库的签名。我照做，运行`sudo apt-key add ...`，结果系统报错：

```bash
sudo: 'apt-key': command not found
```

原来，我用的Ubuntu 26.04是比较新的版本，`apt-key`这个命令已经被彻底移除了，所以无法用老办法添加密钥。

## 第三步：用新的keyring包解决签名问题

既然`apt-key`已废弃，NVIDIA官方也提供了新的方式——安装一个叫`cuda-keyring`的包，它可以自动配置好签名密钥。

我从NVIDIA官网下载了适用于WSL的keyring包：

```bash
wget https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/cuda-keyring_1.0-1_all.deb
```

然后用`sudo dpkg -i`安装它。这次安装很顺利，没有任何报错。接着我执行`sudo apt-get update`，虽然本地旧仓库因为密钥问题还有警告，但NVIDIA官方仓库已经可以正常访问了。

## 第四步：安装CUDA 13.3
更新源后，我运行`sudo apt-get -y install cuda`。这个命令会安装当前最新版的CUDA（当时是13.3）。系统自动下载了大约3.2GB的依赖包，包含编译器、库、工具等，全部安装成功。

安装完成后，我以为万事大吉，于是输入`nvcc -V`想看看版本，结果却提示：
```bash
Command 'nvcc' not found, but can be installed with:
sudo apt install nvidia-cuda-toolkit
```

原来，安装`cuda`这个元包并没有把`nvcc`放到PATH里，或者它安装的是较新版本，但`nvcc`可能不在默认路径。我检查了`/usr/local/cuda`，确实有，但环境变量没配好。不过系统给出了一个简单建议：直接安装`nvidia-cuda-toolkit`这个包。

## 第五步：安装nvidia-cuda-toolkit补齐nvcc
我听取了建议，运行`sudo apt install nvidia-cuda-toolkit`。这个包是Ubuntu官方仓库里的，版本是12.4，它会安装一套完整的CUDA工具链，包括`nvcc`、调试器、性能分析工具等。下载量约2.1GB，安装完成后，再次输入`nvcc -V`，这次成功显示了版本信息：

```bash
nvcc: NVIDIA (R) Cuda compiler driver
Copyright (c) 2005-2024 NVIDIA Corporation
Built on Thu_Mar_28_02:18:24_PDT_2024
Cuda compilation tools, release 12.4, V12.4.131
```

至此，`nvcc`已经可以正常使用了。

## 第六步：验证GPU驱动
最后，我运行`nvidia-smi`检查GPU状态。输出显示我的RTX 3060笔记本GPU已经被识别，驱动版本为566.07，CUDA版本支持为12.7。这说明WSL成功调用了Windows的NVIDIA驱动，不需要在Linux里额外安装驱动。

## 总结
整个过程虽然有点绕，但总结下来就是：

- 1.WSL中不需要安装NVIDIA驱动，直接使用Windows的驱动即可。
- 2.旧版`apt-key`方法已失效，新版Ubuntu必须使用`cuda-keyring`包来添加仓库签名。
- 3.安装cuda元包后，可能nvcc仍不可用，此时可以再安装`nvidia-cuda-toolkit`（Ubuntu官方源）来补齐。
- 4.最终环境验证：`nvcc -V`和`nvidia-smi`都正常，可以开始写CUDA代码了。

这次配置让我明白了新老工具链的差异，也熟悉了WSL下GPU环境的特点。记录下来，希望给同样在Windows上配CUDA的小伙伴一些参考。
