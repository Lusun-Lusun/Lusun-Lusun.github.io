解决VSCode中C++编译报错“undefined reference”：将gcc改为g++

May 10, 2026

记录一次在VSCode中编译C++代码时遇到的链接错误及解决方法

问题现象#

我写了一个简单的C++程序，用到了vector、cin、cout等标准库功能。在命令行中使用g++编译运行一切正常，但是换到VSCode中按Ctrl+Shift+B构建时，却报了一大堆链接错误：

undefined reference to operator new(unsigned long long)' undefined reference tostd::cout'
undefined reference to `std::basic_ostream<char...>& std::endl<char...>(std::basic_ostream<char...>&)'
...

错误信息显示找不到operator new、标准输入输出流等符号，明显是链接阶段出了问题。

原因分析#

VSCode的C++项目默认会生成一个.vscode/tasks.json文件来定义编译任务。我检查了一下发现，该文件中的"command"字段写的是gcc，而不是g++。

  • gcc 是C语言编译器驱动，不会自动链接C++标准库（如libstdc++），因此找不到operator new、std::cout等C++特有的符号。
  • g++ 是C++编译器驱动，会自动链接C++标准库，适合编译.cpp文件。

命令行中我使用的是g++，所以正常；而VSCode调用了gcc，自然就报链接错误了。

解决方法#

有两种方式可以解决，我采用了最简单的一种：通过VSCode的图形菜单重新生成正确的tasks.json。

步骤#

  1. 在VSCode中打开你的C++源文件所在的文件夹。
  2. 点击顶部菜单栏 终端(Terminal) → 配置默认生成任务(Configure Default Build Task…)。
  3. 在弹出的列表中选择 C/C++: g++ build active file（注意一定要选包含g++的选项，不要选gcc）。
  4. VSCode会自动在.vscode文件夹下生成新的tasks.json文件，并询问是否覆盖，点击“允许”。
  5. 再次按Ctrl+Shift+B构建，编译成功，生成的可执行文件可以正常运行。

备选方案：手动修改#

如果你希望手动修改现有的tasks.json，只需将"command"字段的值从"gcc"改为"g++"，保存即可。例如：

{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "build",
            "type": "shell",
            "command": "g++",   // 原来是 gcc，改成 g++
            "args": ["-g", "${file}", "-o", "${fileDirname}/${fileBasenameNoExtension}.exe"],
            "group": {"kind": "build", "isDefault": true}
        }
    ]
}

总结#

  • VSCode的默认C++编译任务可能使用gcc，导致C++程序链接失败。
  • 解决要领：确保编译任务使用的是g++而不是gcc。
  • 可以通过菜单重新选择“g++ build active file”快速修复，也可以手动修改tasks.json。 一个小小的配置差异，浪费了我不少排查时间。记录下来，希望能帮到遇到同样问题的你。
